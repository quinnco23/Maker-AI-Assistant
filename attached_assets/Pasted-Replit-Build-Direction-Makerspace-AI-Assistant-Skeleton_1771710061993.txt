Replit Build Direction: Makerspace AI Assistant (Skeleton + Guided Demo)
Goal

Build a multi-tenant SaaS skeleton with 3 core surfaces:

Landing Page (marketing)

Member Dashboard (students/users)

Admin Dashboard (professors/managers)

The build should be a guided demo with mock data and non-functional placeholders where needed. The focus is UI/UX + structure, not full functionality.

1) UI/UX Requirements (make it feel modern)

Use a modern SaaS app-shell: left sidebar + top bar, responsive, collapsible sidebar on mobile.

Use card-based layouts, lots of whitespace, rounded corners, subtle shadows.

Include dark mode toggle (even if simple).

Include command palette UI (⌘K) stub for navigation/search.

Build strong empty states with CTAs (“Upload your first SOP to enable assistant guidance”).

Accessibility baseline:

keyboard navigable

semantic headings

ARIA labels on buttons/inputs

include Plain-Language Mode toggle in Member top bar (it can simply change copy to simpler text).

6) Data Layer (Minimal + Mock)

but do not spend time on real integrations.

7) Demo Script (must be possible in the skeleton)

Make sure the skeleton supports this click-through:

Admin completes Setup Wizard → uploads SOP → publishes

Member opens Tools Library → sees published SOP

Member starts session → picks “unknown material” → sees STOP + escalation summary

Admin opens Escalations inbox → sees the new handoff packet

8) Definition of Done (Skeleton Phase)

Landing page looks polished

Member dashboard and admin dashboard are fully navigable

Tools Library + one “deep” tool page exists (Laser recommended)

Start Session stepper works visually and can trigger escalation summary

Admin setup wizard + KB upload UI + safety rules editor exist

Escalation inbox exists on admin side

Everything is modern, responsive, and presentable
